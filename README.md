# Projektplanering
Mini-Portfolio-projekt för att visa för framtida arbetsgivare vad ni har lärt er på kursen. Vi lämnar det rätt fritt här, men hitta en balans i att ha med det ni vill visa, men samtidigt att ni behärskar tekniken och inte kör copy-paste från AI eller vibe kodar hela grejen. Tänk på att det inte är så lång tid heller så avgränsa er till vad som är rimligt. Sen kan ni gärna fortsätta på egen hand med fler funktioner senare såklart. 

## Grundkrav
* Projekt med Next.js
* Ska planeras och godkännas innan genomförande
* GitHub Projects eller liknande för hantering av sprintar
* Tydliga commit meddelanden som andra kan förstå och tolka
* “Scrum”-liknande arbetssätt

## Tidsåtgång
2 veckor inklusive planering och redovisning.

## Valbart om ni vill
* Arbeta gärna i feature branches och PR för att öva på det
* Arbeta i grupp för stöd vid PR eller planering osv
* Koppling till externt API eller databas
* Använda bibliotek som shadcn, Tailwind, react-hot-toast, Prisma eller liknande
* Deployment till Vercel eller dylikt

## Planeringsstadie
Ska redovisas för lärare och grupp innan arbetet med kod börjar. Det viktiga här är inte om nedanstående är perfekt utan mer att ni har en klar tanke med vad ni vill åstadkomma med dessa två veckor. 

Planeringen bör innehålla:
* Wireframe/designskiss (gör inget om ni kopierar en grund från någon annanstans, viktigaste är att ni har en design att jobba mot)
* Val av API, db, endpoints, annan extern data i detta.
* “Product Backlog” i er planeringsyta. Sorterad efter prioritering, med funktionalitet/features för projektet

# Tips, förslag och exempel
* Ni får gärna fortsätta att brodera ut webshop-admin om ni vill det. Vi kommer troligen jobba vidare med butikens framsida senare i grupp, så det kan ni gärna avvakta med.
* Jobba vidare med något tidigare projekt från kursen ni vill göra klart, eller något ni vill göra om. Typ pokemons, futurama, filmer eller vad ni nu gjort
* Någon längre och större projekt-tutorial från t ex youtube eller dylikt
* Om ni önskar något helt annat än vad som vi jobbat med, kolla gärna lite med oss innan så vi kan kolla om det är rimligt/lämpligt.

# Redovisning
Innan ni drar igång själva redovisningen - se till att skicka ut länken till ert projekt i chatten (och se till att alla kommer åt) och om ni vill får ni gärna ha en live-share-länk också. 

Ni kommer ha ca 10 minuter på er att:
* Ge en demo på er app där ni delar skärm och visar. Inklusive felhantering, sökningar eller annat som kan vara intressant att visa upp. Ingen kod utan bara funktionalitet i webbläsaren. Däremot får ni gärna berätta kortfattat hur ni löst det tekniskt utan att gå in på för många detaljer.
* Visa/berätta hur ni strukturerade er readme-fil på GitHub för att bäst kommunicera ert projekt.
* Berätta om något som ni tyckte var extra lärorikt/spännande/roligt med utvecklingen av er app.
